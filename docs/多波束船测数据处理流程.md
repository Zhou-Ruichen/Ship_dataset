# 多波束船测数据处理流程

> **项目路径**: `/mnt/data2/00-Data/ship/`
> **生成日期**: 2026-05-05
> **文档目的**: 全面总结当前多波束船测数据从原始文件到最终模型验证的完整处理流程

---

## 1. 项目总览

本项目旨在将全球多波束测深原始数据经过标准化、质量控制、空间聚合、偏差分析、质量分级等一系列步骤，构建高可信的船测验证基准，用于评估全球海底地形模型（如 GEBCO、ETOPO、SRTM15 等）以及 SWOT 卫星反演海底地形的精度。

### 1.1 数据来源

| 数据集 | 路径 | 说明 |
|--------|------|------|
| NCEI 多波束 | `jamstec/multibeam/` | 主数据集，来自 NCEI 的全球多波束测深档案。格式审计扫描 5,140 个 `.dat` 文件，其中 5,096 个为可用测深格式（3-col 5,072 + 6-col 24），最终产出 5,083 个标准化 Parquet |
| NCEI 单波束 | `ncei/` | 辅助数据集，已合并为 `singlebeam.xyz`（lon lat depth 格式） |
| JAMSTEC | `jamstec/` | 日本海洋研究开发机构的重力和水深数据（尚未处理） |

### 1.2 技术栈

- **语言**: Python 3
- **数据格式**: Parquet（中间产物）、TSV/CSV（报告）、NetCDF/NPZ（地形模型）
- **核心库**: pandas, numpy, pyarrow, xarray, scipy
- **机器学习**: scikit-learn, XGBoost, LightGBM（用于残差校正基线训练）

## 3. 各步骤详细说明

### Step 00: 递归解压提取（数据预处理）

**脚本**: `code/03_recursive_extract.py`

> **注意**: 此步骤编号为 03 是因为历史命名原因，但在处理流程中是最先执行的预处理步骤。

**功能**: 从 `raw/subzips/` 中递归解压所有 sub-zip 文件到 `raw/dat_by_subzip/`，处理嵌套 zip 结构。

**前置步骤**:
1. 使用 `bash code/01_extract_subzips.sh` 解压两个大型档案到 `raw/subzips/`（约 776 个 sub-zips）
2. 使用 `bash code/02_check_subzips.sh` 校验完整性，损坏文件移至 `raw/subzips_bad/`（13 个坏文件）

**输入**: `raw/subzips/*.zip`（776 个 sub-zips）

**输出**: `raw/dat_by_subzip/<subzip_name>/**/*.dat`（5,140 个 `.dat` 文件）

**关键逻辑**:
- 对每个 sub-zip，提取到以 subzip 名称命名的子目录
- 检测并递归处理嵌套 zip（如 `.dat.zip`），提取后删除中间 zip 文件
- 提取失败的 sub-zip 移至 `raw/subzips_bad/`
- 输出目录结构: 按日期命名（如 `20120408/`）或按航次命名（如 `KM16-08_leg1_bathymetry_dmo/`）

**运行方式**:
```bash
# 需要先完成前置的 shell 脚本步骤
bash code/01_extract_subzips.sh
bash code/02_check_subzips.sh
python3 code/03_recursive_extract.py
```

### Step 02: 标准化为 Parquet 点表

**脚本**: `code/02_standardize_multibeam_xyz.py`

**功能**: 将每个 `.dat` 文件转为统一 schema 的 Parquet 点表。

**输入**: `raw/dat_by_subzip/` + `manifests/file_manifest.parquet`

**输出**: `derived/points_raw/<file_id>.parquet`（5,083 个文件，共 86 GB）

**点表 Schema**:

| 列名 | 类型 | 说明 |
|------|------|------|
| `file_id` | string | 外键，关联 file_manifest |
| `point_index_in_file` | int64 | 文件内行号（0-based） |
| `lon_raw` | float64 | 原始经度（可能为 0-360 范围） |
| `lat_raw` | float64 | 原始纬度 |
| `lon` | float64 | 标准化经度 `[-180, 180)` |
| `lat` | float64 | 纬度 |
| `depth_m_positive_down` | float64 | 深度（正向下，单位米） |
| `elev_m` | float64 | 高程（= -depth，海平面以下为负） |
| `date_raw` | string | 日期（仅 6-col 文件） |
| `time_raw` | string | 时间（仅 6-col 文件） |
| `sonar_idx` | int64 | 声呐波束编号（仅 6-col 文件） |

**关键设计决策**:
- 经度归一化: `((lon_raw + 180) % 360) - 180`
- 每个 `.dat` 文件对应一个 Parquet，支持断点续传
- 不在点表中冗余存储元数据，通过 `file_id` 关联 manifest

#### Step 02a: 筛选/去重清单

**脚本**: `code/02a_curate_points_manifest.py`

**功能**: 在 file_manifest 基础上增加文件角色标记、include/exclude 标记、重复文件候选检测。

**输出**: `manifests/file_manifest_points_raw.parquet`

**目的**: 标记 transit（过境数据）和重复文件，在后续处理前排除。

### Step 03: 质量控制标记

**脚本**: `code/03_qc_multibeam_points.py`

**功能**: 对每个测深点进行质量控制标记，不删除数据，只添加布尔标记列。

**输入**: `derived/points_raw/*.parquet`

**输出**: `derived/points_qc/<file_id>.parquet`（5,083 个文件，共 86 GB）

**QC 标记列**:

| 列名 | 检查内容 | 通过条件 |
|------|---------|---------|
| `qc_valid_lon` | 经度范围 | `lon ∈ [-180, 180)` |
| `qc_valid_lat` | 纬度范围 | `lat ∈ [-90, 90]` |
| `qc_depth_positive` | 深度正值 | `depth > 0` |
| `qc_depth_not_extreme` | 深度极值 | `depth ≤ 12000m` |
| `qc_elev_negative` | 高程为负 | `elev < 0` |
| `qc_no_nan` | 关键列非空 | 所有核心列非 NaN |
| `qc_pass_basic` | 综合通过 | 以上全部通过 |

**设计原则**: 原始数据存在坐标错误、零深度、极端值等问题，这些会污染后续聚合，因此通过 QC 标记筛选有效点。

### Step 04a: 文件级 1min Cell 聚合

**脚本**: `code/04a_make_multibeam_file_cells.py`

**功能**: 将每个文件的 QC 通过点聚合为 1 弧分（≈1 海里）网格单元统计量。

**输入**: `derived/points_qc/*.parquet`

**输出**: `derived/file_cells_1min/<file_id>.parquet`（5,083 个文件，共 486 MB）

**Cell 定义**:

```
cell_deg = 1/60° (1 弧分)
lon_bin = floor((lon + 180) / cell_deg)
lat_bin = floor((lat + 90) / cell_deg)
cell_id = "1min_{lat_bin}_{lon_bin}"
lon_center = -180 + (lon_bin + 0.5) * cell_deg
lat_center = -90 + (lat_bin + 0.5) * cell_deg
```

**每个 (file, cell) 组合的统计量**: median_depth, mean_depth, std, count, range 等。

---

### Step 04b: 全局 Cell 合并

**脚本**: `code/04b_merge_multibeam_cells.py`

**功能**: 将 5,083 个文件级 cell 合并为全局 1min cell grid。

**输入**: `derived/file_cells_1min/*.parquet`

**输出**: `derived/cells_1min/cells.parquet`（181 MB）

**核心算法: File-balanced Median**

```
对每个 cell:
  1. 收集所有覆盖该 cell 的文件
  2. 每个文件取其 median_depth
  3. 对所有文件 median 取中位数 → cell 的最终深度
```

**为什么这样设计**: 不同文件在同一位置的点密度差异极大。例如一条航迹密集经过某区域 vs 偶尔经过，简单取所有点的 median 会被点密集的文件主导。File-balanced median 确保每个文件权重相等。

**同时记录的辅助统计量**: 点加权均值、文件间标准差/IQR、覆盖文件数等。

### Step 05: 重叠偏差分析

**脚本**: `code/05_analyze_cell_overlap_bias.py`

**功能**: 分析多文件重叠 cell 中的残差和系统性偏差，识别存在问题的文件和航次。

**输入**: `derived/file_cells_1min/*.parquet` + `derived/cells_1min/cells.parquet`

**输出**: `derived/overlap_bias_1min/` 目录（139 MB）

| 文件 | 说明 |
|------|------|
| `overlap_file_cell_residuals.parquet` | 每个 (file, cell) 的残差（123 MB） |
| `file_bias_summary.parquet` | 每个文件的偏差统计 |
| `cruise_bias_summary.parquet` | 每个航次的偏差统计 |
| `suspicious_files.tsv` | 可疑文件清单 |
| `suspicious_cruises.tsv` | 可疑航次清单 |

**核心逻辑**: 在 `n_file_cells ≥ 2` 的 cell 中，计算每个文件 cell 相对于 file-balanced median 的残差，按文件/航次聚合，识别系统性偏差。

**脚本**: `code/07_prepare_primary_validation_cells_1min.py`

| Tier | 条件 | 说明 | 数量 | 占比 | 验证权重 |
|------|------|------|-----:|-----:|---------:|

### Step 08: 验证全球海底地形模型

**脚本**: `code/08_validate_gridded_products_against_ship_cells.py`

**功能**: 用船测 cell 验证全球海底地形模型（gridded bathymetry products）。

**配置文件**: `configs/gridded_products_validation.yaml`

**输入**:
- `derived/validation_cells_1min/primary_ship_validation_cells_1min.parquet`
- 配置文件中指定的 5 个全球模型 + 1 个 SWOT 区域模型

**验证的模型产品**:

| 产品 | 分辨率 | 文件大小 | 采样方法 |
|------|--------|---------|---------|
| GEBCO_2024 | 15s | 7.0 GB | cell_median → fallback center_nearest |
| ETOPO_2022 | 1min | 469 MB | center_bilinear |
| SRTM15_V2.7 | 15s | 6.2 GB | cell_median → fallback center_nearest |
| SDUST_2023 | 15s | 535 MB | cell_median（0-360 经度） |
| TOPO_25.1 | 15s | 523 MB | cell_median → fallback center_nearest |
| SWOT_T1 | 区域 | NPZ | center_bilinear（仅 T1 足迹范围） |

> **当前状态**: 已完成 T1 footprint 区域的 8,121-cell 验证（5 个全球产品 + SWOT_T1），全局 2,394,115-cell 的完整验证仍需单独执行。

**采样策略说明**:

| 方法 | 说明 | 适用场景 |
|------|------|---------|
| `cell_median` | 聚合 cell 内所有产品像素取中位数 | 高分辨率产品（15s） |
| `cell_mean` | 聚合 cell 内所有产品像素取均值 | 高分辨率产品（对离群值敏感） |
| `center_bilinear` | 在 cell 中心点双线性插值 | ~1min 分辨率产品 |
| `center_nearest` | 最近邻采样 | 快速验证 |

**z 轴约定**:
- `elevation_negative_ocean`: 海洋为负值（如 GEBCO、ETOPO）
- `depth_positive_down`: 海洋为正值（脚本自动取负转换）

**已知问题**: SDUST_2023 有陆地掩膜问题（6% 的海洋 cell 被标记为陆地），导致 bias=389m。

**输出文件** (`derived/model_validation_1min/`):

| 文件 | 说明 |
|------|------|
| `validation_by_cell_<product>.parquet` | 逐 cell 误差 |
| `validation_metrics_summary.parquet/.tsv` | 总体指标 |
| `validation_metrics_by_quality_tier.*` | 按质量分层指标 |
| `validation_metrics_by_depth_bin.*` | 按深度分层指标 |
| `validation_metrics_by_region_10deg.*` | 按 10°×10° 区域分层指标 |
| `model_validation_report.md` | 可读汇总报告 |

**全量运行估算**: ~3.5-4 小时，输出 ~0.6-1.3 GB。

```bash
python3 code/08_validate_gridded_products_against_ship_cells.py \
  --config configs/gridded_products_validation.yaml \
  --overwrite

# 仅验证特定产品
python3 code/08_validate_gridded_products_against_ship_cells.py \
  --config configs/gridded_products_validation.yaml \
  --product-name GEBCO_2024 --overwrite

# 公平对比：将所有产品限定在 SWOT_T1 的足迹范围内
python3 code/08_validate_gridded_products_against_ship_cells.py \
  --config configs/gridded_products_validation.yaml \
  --comparison-footprint SWOT_T1 --overwrite
```

### Step 10: 构建船测监督残差数据集

**脚本**:
- `code/10_build_ship_supervised_residual_dataset_T1.py` — 构建残差数据集
- `code/10b_generate_stratified_spatial_splits_T1.py` — 生成分层空间划分

**功能**: 将船测 cell 与多个地形模型产品的高程值合并，构建监督学习用的残差数据集。

**核心概念**: 
- **残差 (residual)** = `ship_elev - model_elev`（等价于 `model_depth - ship_depth`）
- 正值 = 模型偏深，负值 = 模型偏浅
- 用于训练模型学习如何校正地形模型的系统性误差

**输入**: 
- Step 08 的 validation_by_cell parquet（6 个产品）
- 船测验证 cell 主表

**输出**: `derived/ship_supervised_residual_T1/`

| 文件 | 说明 |
|------|------|
| `ship_residual_dataset_T1.parquet` | 完整残差数据集（8,121 个 cells，6 个产品高程列均无缺失） |
| `ship_residual_dataset_T1_train.parquet` | 初始训练集（1° block split, legacy） |
| `ship_residual_dataset_T1_val.parquet` | 初始验证集 |
| `ship_residual_dataset_T1_test.parquet` | 初始测试集 |
| `splits/block025_stratified_seed42/` | **最终使用的分层空间划分**（0.25° block, 113 blocks） |
| `ship_residual_dataset_report.md` | 数据集构建报告 |

**最终使用的空间划分** (`block025_stratified_seed42`):

| split | n_cells | 占比 | n_blocks |
|-------|--------:|-----:|---------:|
| train | 4,873 | 60.0% | 52 |
| val | 1,624 | 20.0% | 30 |
| test | 1,624 | 20.0% | 31 |

**目标变量**: `target_residual_SWOT_T1` = `ship_elev - SWOT_T1_elev`（等价于 `SWOT_T1_depth - ship_depth`）

### Step 11: 残差基线模型训练

**脚本**: `code/11_train_residual_baselines_T1.py`

**功能**: 训练多种残差校正基线模型，评估不同特征集和算法的效果。

**特征集**:

| 特征集 | 说明 | 可用于真实推理？ |
|--------|------|-----------------|
| `swot_only_inference` | 仅使用 SWOT 预测及其派生特征 | ✅ 是 |
| `global_product_fusion_diagnostic` | 使用多个全球产品高程值 | ⚠️ 诊断用 |
| `oracle_ship_quality_diagnostic` | 使用船测质量信息（不可实际获取） | ❌ 仅诊断上限 |

**模型类型**:
- 无校正 (no_correction)
- 全局偏差校正 (global_bias_correction)
- 深度分箱偏差校正 (depth_bin_bias_correction)
- 线性回归 (LinearRegression)
- 岭回归 (Ridge)
- 随机森林 (RandomForest)
- XGBoost / LightGBM

**关键结论**:

| 方法 | Test RMSE | 说明 |
|------|-----------|------|
| 无校正 | 160.37 m | SWOT_T1 原始精度 |
| 最佳 SWOT-only (XGBoost/LightGBM) | 141.18 m | 相对降低 12.0% |
| 全局产品融合 (Linear) | 29.87 m | 诊断上限（需其他全球产品） |
| 深度依赖校正 | 157.17 m | 改善有限（2.0%） |
| 空间/非线性校正 | 141.18 m | 主要改善来源 |

**输出**: `derived/ship_supervised_residual_T1/baselines_block025/`

## 7. 关键设计决策总结

| 决策 | 选择 | 原因 |
|------|------|------|
| 数据格式 | Parquet | 列式存储高效，支持按文件分片，支持断点续传 |
| QC 策略 | 标记不删除 | 保留完整数据，下游按需筛选 |
| Cell 合并算法 | File-balanced median | 防止高密度文件主导结果 |
| Cell 分辨率 | 1 弧分（≈1.85 km） | 平衡精度与数据量 |
| 质量分层 | A/B/C tier + exclude | 分级利用数据，而非一刀切 |
| 空间划分 | 0.25° block | 防止空间数据泄漏 |
| 残差定义 | `ship_elev - model_elev`（= `model_depth - ship_depth`） | 正值 = 模型偏深，负值 = 模型偏浅 |
| 模型验证 | 多层次（全局/区域/深度/质量） | 全面理解模型性能 |
