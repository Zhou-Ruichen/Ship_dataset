# Backend Development Guidelines

> Conventions for the Python data-processing pipeline that lives under
> each dataset's `code/` directory.
>
> This is **not a typical web backend** — there is no HTTP layer, no ORM,
> no authentication. "Backend" here means the offline batch pipeline that
> turns raw multibeam `.dat` files into validated bathymetry products.

---

## Directory provenance — JAMSTEC (formerly mislabelled as "NCEI_multibeam")

`ship/jamstec/multibeam/` holds bathymetry data entirely from JAMSTEC
(R/V Kaiyo / Kairei / Mirai, 2000–2015). This directory was originally
named `ship/NCEI_multibeam/` because the upstream archive
(`国外水深第一/第二部分.zip`) was incorrectly labelled NCEI;
investigation in 2026-05 confirmed JAMSTEC provenance and the directory
was renamed to its current location as part of task
`05-11-singlebeam-integration` (PR-A, 2026-05-16). Sibling sub-corpora
under `ship/jamstec/` (`gravity_data/`, `archive/`) are the same
JAMSTEC source in different forms.

The string literal `source_dataset = "NCEI_multibeam"` still appears in
3 scripts under `jamstec/multibeam/code/` as a logical lineage label
(not a path) — it is load-bearing for Step 08 bit-identical
verification. See `docs/experiments/2026-05_dataset-source-attribution.md`
for the full evidence chain.

## How to read this directory

Read in this order when you're new (or when an AI sub-agent is starting
work in `<dataset>/code/`):

1. [`directory-structure.md`](./directory-structure.md) — where everything lives.
2. [`quality-guidelines.md`](./quality-guidelines.md) — required script structure, run-label gating, forbidden patterns.
3. [`pipeline-design-decisions.md`](./pipeline-design-decisions.md) — *why* the pipeline is shaped this way.
4. [`data-contracts.md`](./data-contracts.md) — schemas every stage depends on.
5. [`quality-tiering.md`](./quality-tiering.md) — the file-level flags and cell-level A/B/C tiers.
6. [`error-handling.md`](./error-handling.md) — per-file failure pattern.
7. [`logging-guidelines.md`](./logging-guidelines.md) — stdlib `logging` conventions.
8. [`database-guidelines.md`](./database-guidelines.md) — parquet / manifest / TSV / YAML storage conventions (no actual DB).
